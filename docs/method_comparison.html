Key Takeaway: Our Unique Position

  • vs Brain-JEPA: We operate on EEG (ms resolution, not seconds). We have causal prediction (not bidirectional fill-in). We add subject adversarial training. Brain-JEPA has better positional encoding (gradient positioning) and much larger pretraining scale.
  • vs EchoJEPA: Completely different domain (brain vs heart). We share the JEPA paradigm but add causality and neuroscience priors. EchoJEPA has massive data scale (18M samples) and model scale (1.1B).
  • Our unique story: The only model that combines causal temporal prediction + spatial region masking + ms-level resolution. If oddball PE analysis works, we demonstrate something neither fMRI nor echo models can: linking computational prediction errors to neural ERPs.

Part 2: BrainWM v3 Pipeline

1

Raw EEG Input

[B, 1024, 64] — 4s × 256Hz × 64 channels
Why 4s/256Hz: PhysioNet MI dataset standard. 4s covers a complete motor imagery trial. 256Hz captures up to 128Hz (Nyquist) — sufficient for all classical EEG bands (delta through high-gamma).
2

EEG Augmentation

EEGAugmentation
[B, 1024, 64] → [B, 1024, 64]
Generic: time shift (±25ms), amplitude scaling, Gaussian noise, channel dropout
EEG-specific (v3): electrode pop artifact, 50/60Hz powerline, slow baseline drift
Why: Borrowed from EchoJEPA’s domain-specific augmentation philosophy. Makes representations robust to real-world EEG artifacts. The model learns to ignore noise and focus on neural signal.
Advantage: Model trained on augmented data is robust to real clinical EEG noise without explicit preprocessing.
3

Channel Encoding

LearnableFilterBank → TemporalAttention → MLP
[B×40×64, 26] → [B×40×64, 128]
Each 100ms window of each channel → 128-dim latent vector
FilterBank (5 bands): Initialized as delta/theta/alpha/beta/gamma bandpass filters, but learnable. Decomposes EEG into frequency components — core of EEG analysis.
Temporal Attention (2 layers): Learns cross-frequency-time interactions within each 100ms window. Captures patterns like event-related spectral perturbation (ERSP).
Why channel-independent: Each channel processed independently ensures the model can handle variable electrode montages. Cross-channel interaction happens in the next stage.
Advantage: Frequency decomposition is built into the architecture, not learned from scratch. Attention captures nonlinear spectro-temporal features.
4

Regional Cross-Attention Aggregation

RegionalAttentionAggregation
[B×40, 64, 128] → [B×40, 5, 128]
64 channel latents → 5 region latents (frontal, central, parietal, temporal, occipital)
Design: Learnable query per region + cross-attention over channels belonging to that region (based on 10-20 system mapping).
Why regions: Brain functions are organized by region (motor cortex = central, visual = occipital). Grouping by anatomy introduces neuroscience prior without losing flexibility (attention weights are learned, not fixed).
Why cross-attention: Different channels contribute differently to a region’s state depending on context. Attention learns dynamic channel weighting.
Advantage: Interpretable spatial structure. Region-level prediction errors can be mapped to known neuroanatomy (e.g., frontal PE = MMN, parietal PE = P300).
5

Brain State Composition

BrainStateComposer (concat + region/temporal embedding)
[B, 40, 5, 128] → [B, 40, 640]
5 region latents concatenated → brain state vector + position embedding
Region embedding: Learnable vector per region, added before concat. Gives each region a unique “identity” in the joint space.
Temporal embedding: Position embedding per timestep (0-39). Only added to online encoder, NOT to EMA target. This prevents the contrastive loss from being solved by position matching.
Content-only targets: EMA encoder returns states without temporal position — forces InfoNCE to rely on actual EEG content, not position tags.
Advantage: Asymmetric design (online has position, target doesn’t) is key to preventing shortcut solutions in contrastive learning.
5b

Region Masking v3 NEW

RegionMaskPredictor (cross-attention)
Mask 1-2 of 5 regions → predict masked from unmasked
[B×40, R_unmasked, 128] → [B×40, R_masked, 128]
Inspired by: Brain-JEPA’s Cross-ROI masking strategy.
How: 50% probability during training. Randomly select 1-2 regions to mask. Replace their slices in brain_states with learnable mask tokens. RegionMaskPredictor uses cross-attention: masked region queries attend to unmasked region latents.
Key distinction from Brain-JEPA: We only mask spatially, NOT temporally. The Mamba world model remains strictly causal. Region masking asks “can you infer parietal activity from frontal+temporal at the same moment?” — this is functional connectivity, not temporal prediction.
Two effects: (1) RegionMaskPredictor learns inter-region relationships directly. (2) World model receives spatially-incomplete states, forcing it to use temporal context to compensate.
Advantage: Adds spatial learning objective orthogonal to temporal prediction. Strengthens both spatial and temporal representations.
6

Causal Mamba World Model

CausalMambaWorldModel (6 layers, d=512)
[B, 40, 640] → [B, 40, 512]
Brain state sequence → causal hidden state sequence
Why Mamba (SSM): State space models are naturally causal: h(t+1) = A·h(t) + B·x(t). Information flows strictly forward in time. Unlike Transformer attention which is bidirectional by default.
Why not Transformer: Causal masking in Transformers works but is a constraint on a bidirectional architecture. Mamba’s causality is structural — there’s no way for future to leak into past. This is conceptually aligned with predictive coding theory (the brain predicts forward, not backward).
Selective scan: Input-dependent gating (B, C, delta parameters) lets the model selectively attend to or ignore parts of the sequence, analogous to neural gain control.
Advantage: Enables autoregressive rollout (imagination). The model can generate future brain states step-by-step, something bidirectional models cannot do.
7

Multi-Horizon Prediction Head

PredictionHead (MLP + horizon/position embedding)
[B, 40-k, 512] → [B, 40-k, 640] for each k ∈ {1, 3, 5}
Horizon embedding: Tells the head which future step to predict (k=1, 3, or 5). Shared MLP, different embeddings per horizon.
Position embedding: Tells the head the target’s absolute position (not in target, only in prediction path). Helps distinguish which specific timestep to generate.
Why k=1,3,5: k=1 (100ms) captures immediate dynamics. k=3 (300ms) captures ERP timescale (P300). k=5 (500ms) captures slower cognitive processes. Harder horizons force deeper temporal understanding.
Advantage: Multi-scale temporal prediction. k=1 learns fast dynamics, k=5 learns slow dynamics. Brain processes span multiple timescales.
8

EMA Target Encoder

EMAEncoder (momentum 0.996 → 0.9999)
Exponential moving average copy of BrainStateComposer
Why EMA: Provides stable prediction targets (BYOL/JEPA paradigm). Without EMA, the target would change as fast as the online encoder, making the prediction task unstable.
Why 0.9999 cap (not 1.0): EMA=1.0 means target freezes completely → online encoder overfits to static target → representation collapse. 0.9999 ensures target always slowly tracks online encoder.
Content-only output: Returns states without temporal position embedding. Critical for contrastive learning — prevents position-matching shortcut.
Advantage: Triple anti-collapse: (1) EMA never freezes, (2) VICReg enforces variance/decorrelation, (3) content-only targets prevent trivial solutions.
9

Subject Adversary

SubjectAdversary (GRL, delayed start)
[B, 640] → [B, 109] subject logits
Gradient Reversal Layer: Classifier tries to identify which subject. GRL flips gradients so encoder learns to confuse the classifier → subject-invariant representations.
Delayed start (v3): α=0 for first 30% of training. Prediction must establish meaningful representations before adversarial pressure is applied. Otherwise GRL drives collapse (making all representations identical is the easiest way to confuse a classifier).
Why needed: EEG has strong individual differences (skull thickness, electrode impedance, brain anatomy). Without adversarial training, representations encode “who” not “what.”
Advantage: Neither Brain-JEPA nor EchoJEPA has explicit cross-subject invariance. Critical for EEG where individual differences dominate.

Part 3: Loss Function Landscape

1. Within-Sequence InfoNCE

L = -log( exp(sim(pred_t, target_t)/τ) / Σ_t' exp(sim(pred_t, target_t')/τ) )
For each prediction at time t, it must correctly identify the target at t+k among all M timesteps in the same trial. Forces temporal discrimination based on EEG content, not position.
Horizons: k=1 (100ms), k=3 (300ms), k=5 (500ms). Weights: [1.0, 0.5, 0.25]
Origin: CPC (van den Oord 2018), adapted for within-sequence contrast

2. Region Masking Loss

L = 2 - 2 · cos(predicted_region, EMA_target_region)
RegionMaskPredictor’s output vs EMA target for masked regions. Negative cosine similarity. Learns inter-region functional connectivity.
Origin: Brain-JEPA Cross-ROI masking, adapted for EEG regions

3. VICReg Variance

L_var = mean( ReLU(1 - std_per_dim) )
Per-dimension standard deviation of brain states must be ≥ 1. Prevents representation collapse where all outputs become the same vector. λ=5.0.
Origin: VICReg (Bardes 2022)

4. VICReg Covariance

L_cov = Σ_{i≠j} C(i,j)² / D
Off-diagonal elements of covariance matrix should be zero. Prevents dimensional collapse where all 640 dimensions encode the same information. λ=1.0.
Origin: VICReg (Bardes 2022)

5. Subject Adversarial

L_adv = CrossEntropy(subject_logits, subject_id)
With gradient reversal (α ramps 0→1.0 from 30%-80% of training). Pushes encoder to remove subject identity from representations. λ=0.1.
Origin: Domain adversarial training (Ganin 2016)

Part 4: Design Decisions to Review

Current Strengths

  • Hybrid causal+spatial: Temporal InfoNCE (causal) + region masking (spatial) learn orthogonal features. Neither Brain-JEPA nor EchoJEPA has this combination.
  • Multi-scale prediction (k=1,3,5): Unique among the three. Captures brain dynamics at multiple timescales.
  • 100ms resolution: Can capture ERPs (MMN ~200ms, P300 ~400ms) that fMRI fundamentally cannot.
  • Causal rollout: Unique capability. Can “imagine” future brain states autoregressively.
  • Triple anti-collapse: EMA + VICReg + content-only targets. More robust than Brain-JEPA/EchoJEPA (which rely only on EMA + stop-gradient).

Potential Concerns / Open Questions

  • Scale gap: 109 subjects vs 40K (Brain-JEPA) / 18M (EchoJEPA). The method may work but results may not compete on benchmarks. Mitigation: focus on analyses that small-scale models can demonstrate (oddball PE, rollout quality) rather than brute-force classification.
  • Position encoding: Our anatomical region grouping is static and coarse. Brain-JEPA’s Brain Gradient Positioning (data-driven functional connectivity gradients) is more principled. Could we compute channel-level functional gradients from training data?
  • Temporal position in prediction head: PredictionHead has position_embedding that tells it WHERE to predict. Since targets have no position, the head can’t trivially match by position. But it could memorize average content at each position across training. Need to monitor k1/k3/k5 during training — if they drop to near-zero quickly, this shortcut may be active.
  • Region masking + world model interaction: When regions are masked, the world model receives degraded states. Scheduled sampling also modifies states. Both happening simultaneously could destabilize training. May need to disable scheduled sampling when region masking is active.
  • No stimulus conditioning: Without external event input, the model predicts brain dynamics purely from internal state. Adding stimulus type (e.g., left/right hand cue) as conditioning would make it a true “world model” and address the “just a predictor” critique.

Recommended Next Steps (Priority Order)

  • 1. Train v3 and verify loss dynamics: k1/k3/k5 should start ~3.5, decrease, with k1<k3<k5. rmask should decrease. var/cov should approach 0.
  • 2. Linear probe evaluation: Freeze encoder, train linear classifier on motor imagery. This is the standard benchmark to prove representation quality.
  • 3. Oddball PE analysis: The killer experiment. If model PE correlates with MMN/P300, this validates the predictive coding story and is beyond reach of any fMRI model.
  • 4. Imagination rollout analysis: Qualitative: do rollout trajectories look like plausible brain dynamics? Quantitative: cosine similarity between imagined and actual future states.
  • 5. (Optional) Stimulus conditioning: Add event type as input to world model. Would upgrade from “predictor” to “world model” and enable counterfactual reasoning.
